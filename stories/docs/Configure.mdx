# Configure your project

TalkingSh*rt Storybook renders production HTML templates with the same CDN token bundle as the live site.

## Production pipeline

CDN tokens load first in `.storybook/preview-head.html`, then `/css/main.css`. Stories import templates from `public/js/components/*.html` via `stories/lib/render-template.js`.

[Storybook configuration docs](https://storybook.js.org/docs/configure)

## Layout context

Stories render production HTML templates directly on Storybook’s default padded canvas.

[Layout docs](?path=/docs/docs-layout--docs)

## Autodocs

Each Figma component has a Docs tab with rendered stories and HTML source. Tags and descriptions live in `stories/figma/*.stories.js`.

[Autodocs guide](https://storybook.js.org/docs/writing-docs/autodocs)

## Publish and share

Production builds copy Storybook to `/storybook/` on the Netlify site. Share that URL with anyone — no local setup required.

```bash
npm run build:site
```

[Publishing docs](https://storybook.js.org/docs/sharing/publish-storybook)

## Commands

| Command | What it does |
|---------|--------------|
| `npm run storybook` | Local dev at localhost:6006 |
| `npm run build:site` | Build Storybook → `public/storybook/` |
| `npm run dev` | Site + Netlify functions |
| `npx vitest` | Browser tests on stories |

## Tokens

All visual values come from the CDN bundle. Components use `var(--theme-*)` and `var(--comp-*)` only. See [Screen sizing](?path=/docs/docs-screensizing--docs) for viewport and breakpoint tooling.
